# 跨 AI 记忆

这是 Memmy 区别于其他工具的核心能力：让记忆在所有 AI 之间流动，而不是锁死在某一家。

## 它是怎么工作的

Memmy 的记忆链路分三步：

1. **扫描采集**：增量读取本地的 AI 对话，自动提炼值得记住的片段
2. **结构化整理**：把零散内容整理成有标签、可检索的记忆，去重排序
3. **按需注入**：无论你切换到哪个 AI，都把最相关的记忆精准送回它手里

## 记得什么

Memmy 关注那些「代码仓库里没有、却最该流通」的知识：

| 类型 | 例子 |
| --- | --- |
| 决策记忆 | 为什么选了方案 A 而不是 B |
| 偏好记忆 | 这个模块偏好什么风格 |
| 经验记忆 | 某个 Bug 的完整排查过程 |
| 上下文记忆 | 产品意图、业务约束 |

## 支持的 AI 工具

目前支持扫描和注入的工具包括：

- Cursor
- Claude Code
- Codex
- 更多工具持续接入中

## 为什么不绑定某一家

记忆存在你自己的设备上，不随平台流失。今天用 Cursor、明天换 Codex，默契始终跟着你走。
